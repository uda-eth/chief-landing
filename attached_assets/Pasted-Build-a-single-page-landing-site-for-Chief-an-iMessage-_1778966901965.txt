PRODUCT POSITIONING
==========================================
Product name: Chief

==========================================
SIGNUP FORM BEHAVIOR — CRITICAL
==========================================
On form submission:

1. Read UTM parameters from the URL on page load and store in component state:
   - utm_source, utm_medium, utm_campaign, utm_content
   - Also capture document.referrer

2. POST to this webhook URL (use fetch with no-cors mode since it's a Google Apps Script):
   https://script.google.com/macros/s/AKfycbyxuWxWeaaXQPNzqBByPpDTheRpKAhd05Tro3Hgw56hLEKrnH6k-mu6VjXh-12VSM7N/exec

3. Payload JSON must include these fields:
{
  "candidate_id": "chief",
  "product_name": "Chief",
  "product_slug": "chief",
  "subdomain": "https://chief.remyndai.com",
  "lead_name": "<form name>",
  "lead_email": "<form email>",
  "lead_company_role": "<form: what kind of work>",
  "intent_signal": "<form: most annoying admin task>",
  "notes_from_lead": "<form: most annoying admin task>",
  "custom_question": "What's the most annoying admin task you'd hand off first?",
  "custom_answer": "<form: most annoying admin task>",
  "source": "<utm_source or 'direct'>",
  "channel": "<utm_medium or 'organic'>",
  "campaign": "<utm_campaign or ''>",
  "utm_source": "<from URL>",
  "utm_medium": "<from URL>",
  "utm_campaign": "<from URL>",
  "utm_content": "<from URL>",
  "referrer": "<document.referrer>",
  "landing_page_url": "<window.location.href>",
  "confirmation_page_url": "https://chief.remyndai.com/thank-you",
  "consent_captured": "Yes"
}

4. Use mode: 'no-cors' on the fetch — Google Apps Script will accept the POST but the browser won't see the response. That's fine. Treat the request as successful if the fetch promise resolves.

5. On success (or fetch resolution), navigate to /thank-you.

6. Show loading state on the button while submitting ("Sending…").

7. Basic client-side validation: require name, email (with regex check), and work description. Disable submit until valid.
